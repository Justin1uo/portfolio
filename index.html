03 / 项目经历

两个完整项目，覆盖后端开发核心场景

🍔 苍穹外卖 — 外卖平台后端系统

2025.12 — 2026.05
Spring Boot MySQL Redis JWT Nginx 阿里云 OSS WebSocket Git

基于 Spring Boot 的前后端分离外卖平台，包含用户端、商家端、管理端三端， 涵盖菜品管理、购物车、订单全流程、用户认证、报表统计等核心功能。

  • 实现管理端 + 商家端完整业务模块，包括员工管理、菜品/套餐 CRUD、分类管理、订单状态机流转（待接单→备餐→派送→完成/取消），保证业务逻辑闭环
  • 基于 Redis 缓存菜品分类数据，利用 @Cacheable / @CacheEvict 注解管理缓存生命周期，将高频查询接口数据库调用量降低约 70%
  • 使用 JWT + 自定义拦截器实现双端（管理员/用户）Token 鉴权，做到接口级权限隔离
  • 集成 阿里云 OSS 实现菜品图片上传，封装统一上传工具类，支持文件类型校验与 URL 回显
  • 采用 WebSocket 推送新订单提醒与催单通知，实现商家端实时消息感知
  • 利用 Apache POI 导出近 30 日营业额、订单量、客单价等运营数据 Excel 报表，支持管理端一键下载
  • 借助 Trae AI Coding 工具辅助编写样板代码与 SQL 语句，调试周期平均缩短约 40%

⭐ 黑马点评 — 同城商铺点评平台（Redis 高并发实践）

2026.05 — 至今
Spring Boot Redis MySQL JWT

基于 Spring Boot + Redis 的本地生活点评平台，重点深入 Redis 在真实业务场景中的核心应用与高并发方案。

  • 基于 Redis String 存储短信验证码（TTL 120s 限制频繁发送），结合 Token 机制替代传统 Session，解决分布式场景下 Session 共享问题；通过拦截器自动刷新 Token 有效期，实现无感续期
  • 系统性解决缓存三大问题：缓存穿透采用空值缓存方案拦截非法 key 直透；缓存击穿同时实践互斥锁（setnx）与逻辑过期两种方案，深入理解强一致与高可用的取舍；缓存雪崩通过随机 TTL 离散化过期时间，避免大批缓存同时失效冲击数据库
  • 秒杀场景下以乐观锁（CAS）解决库存超卖问题；基于 Redis setnx 实现分布式锁原型，进一步引入 Redisson 解决锁的可重入、Watch Dog 自动续约及 Lua 脚本保证释放操作原子性问题
